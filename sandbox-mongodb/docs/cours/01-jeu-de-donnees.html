Cours 01

Présentation du jeu de données

Le cours utilise OpenIntro `nyc`, un dataset de notations Zagat sur 168 restaurants new-yorkais.

Ce que décrit le dataset

  • Nom du restaurant.
  • Prix estimé pour deux personnes.
  • Notes `food`, `decor`, `service` sur 0 à 30.
  • Position est ou ouest de 5th Avenue.

Objectif pédagogique : passer d'un CSV plat à un modèle documentaire exploitable.

Collections issues du réel

`nyc_restaurant_reviews_raw`

Lignes brutes du CSV, utiles pour observer la source et les conversions.

`restaurants`

Document orienté établissement : cuisine, prix, notes, tags.

`reviews`

Notation agrégée orientée avis, reliée par `restaurant_id`.

`neighborhoods`

Collection pédagogique annexe créée à l'initialisation.

Collections générées

`orders`

100 000 commandes simulées pour chiffre d'affaires, filtres et index.

`review_details`

50 000 avis détaillés avec note, sentiment et contexte de visite.

`events`

300 000 événements applicatifs pour analyser l'usage.

Questions métier possibles

  • Quelles cuisines ont les meilleures notes ?
  • Quels restaurants offrent le meilleur rapport note/prix ?
  • Quels tags reviennent le plus souvent ?
  • Quels restaurants cumulent satisfaction élevée et revenus importants ?
  • Quels index accélèrent réellement les requêtes fréquentes ?
