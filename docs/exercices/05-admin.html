TP 05

Administration, sauvegarde et sécurité

Manipuler les commandes d'administration sans intégrer les corrections au support.

Objectifs

  • Vérifier l'état du service MongoDB.
  • Comprendre les utilisateurs et les droits.
  • Réaliser une sauvegarde et une restauration.
  • Identifier les bonnes pratiques de sécurité.

Commandes utiles

docker compose ps
docker compose logs --tail=80 mongodb

docker compose exec mongodb mongosh \
  "mongodb://root:rootpass@localhost:27017/nyc_food?authSource=admin"

Exploration admin

db.stats()
db.restaurants.findOne()
db.getUsers()
show collections

Exercices

  1. Vérifier que MongoDB et Mongo Express sont actifs.
  2. Afficher les statistiques de la base `nyc_food`.
  3. Identifier les collections les plus volumineuses.
  4. Créer un utilisateur en lecture seule sur `nyc_food`.
  5. Tester qu'un utilisateur en lecture seule ne peut pas insérer de document.
  6. Réaliser une sauvegarde avec `mongodump`.
  7. Restaurer la sauvegarde dans une base de test.
  8. Lister les bonnes pratiques de sécurité applicables au sandbox.
