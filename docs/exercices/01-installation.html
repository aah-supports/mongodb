Liste 01

Installation et exploration

Démarrer le sandbox et observer les collections initialisées automatiquement.

Objectifs

  • Démarrer MongoDB avec Docker Compose.
  • Vérifier la création automatique des collections.
  • Se connecter avec Mongo Express et `mongosh`.
  • Identifier les collections et la structure des documents.

Démarrage

docker compose up -d
docker compose ps

Mongo Express : `http://localhost:8083`

docker compose exec mongodb mongosh \
  "mongodb://root:rootpass@localhost:27017/nyc_food?authSource=admin"

Collections créées au démarrage

Au premier lancement avec un volume vide, le conteneur MongoDB charge les collections de base et génère les collections de volume.

  • `nyc_restaurant_reviews_raw` : données brutes du CSV.
  • `restaurants` : vue documentaire orientée restaurant.
  • `reviews` : vue orientée avis/notation.
  • `neighborhoods` : collection pédagogique annexe.
  • `orders`, `review_details`, `events` : collections générées pour les exercices de volume.

Exploration

show dbs
db
show collections

db.restaurants.countDocuments()
db.reviews.countDocuments()
db.nyc_restaurant_reviews_raw.countDocuments()

db.restaurants.findOne()
db.reviews.findOne()
db.nyc_restaurant_reviews_raw.findOne()

Questions

  1. Quels champs sont communs à tous les restaurants ?
  2. Quels champs sont imbriqués ?
  3. Quelles différences observe-t-on entre la collection brute et les collections transformées ?
  4. Quels champs semblent utiles pour filtrer ?
  5. Quels champs semblent utiles pour faire des statistiques ?
