Travailler avec mongosh

Le shell officiel permet d'écrire, reproduire et comprendre précisément les commandes MongoDB.

Connexion au sandbox

docker compose exec mongodb mongosh \
  "mongodb://root:rootpass@localhost:27017/nyc_food?authSource=admin"
  • `root` / `rootpass` : authentification.
  • `nyc_food` : base cible.
  • `authSource=admin` : base d'authentification.

Navigation

show dbs
db
use nyc_food
show collections

Fonctions et types utiles

`ISODate()`

`ISODate()` sert à créer une vraie date MongoDB. Ici, on cherche les commandes créées à partir du 1er juin 2026.

db.orders.find({
  created_at: {
    $gte: ISODate("2026-06-01T00:00:00Z")
  }
})

`ObjectId()`

`ObjectId()` sert à retrouver un document par son identifiant MongoDB. Ici, on cible un document précis avec son `_id`.

db.orders.findOne({
  _id: ObjectId("665f1c2a9b8f3a0012ab34cd")
})

`db.getSiblingDB()`

`db.getSiblingDB()` permet de changer de base sans changer de connexion. Ici, on travaille dans la base `event_booking`.

const eventDb = db.getSiblingDB("event_booking")
eventDb.users.findOne()

`toArray()` et `forEach()`

On transforme le curseur en tableau avec `toArray()`, puis on parcourt les résultats avec `forEach()`. Ici, on affiche trois commandes payées.

const paidOrders = db.orders.find({ status: "paid" }).limit(3).toArray()

paidOrders.forEach(order => {
  print(order.order_id)
})

`distinct()`

`distinct()` sert à lister les valeurs différentes d'un champ. Ici, on récupère les cuisines présentes dans la collection.

db.restaurants.distinct("cuisine")

Lire et filtrer

db.restaurants.countDocuments()
db.restaurants.findOne()
db.restaurants.find().limit(5)
db.restaurants.find(
  { "ratings.overall": { $gte: 22 } },
  { _id: 0, name: 1, cuisine: 1, "ratings.overall": 1 }
).sort({ "ratings.overall": -1 }).limit(10)

Création et import de secours

Dans le parcours normal, les collections sont créées au premier lancement des conteneurs avec un volume vide.

docker compose up -d

Si les collections de base doivent être recréées manuellement, on utilise les commandes `mongoimport --drop` du cours jeu de données. Les apprenants n'ont pas à lancer de script d'import ou de génération dans le parcours standard.
