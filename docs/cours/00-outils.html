Outils : Docker, Atlas et mongosh

Un environnement reproductible pour pratiquer MongoDB sans dépendre de l'installation locale de chaque machine.

Starter : commandes utiles

cd sandbox-mongodb
docker compose up -d
docker compose ps

`up -d` lance MongoDB et Mongo Express en arrière-plan. Au premier lancement avec un volume vide, MongoDB initialise automatiquement les collections du cours.

docker compose exec mongodb mongosh \
  "mongodb://root:rootpass@localhost:27017/nyc_food?authSource=admin"

Cette commande ouvre `mongosh` directement sur la base `nyc_food`.

http://localhost:8083

Cette URL ouvre Mongo Express, l'interface web de vérification.

Mongo Express et imports

Mongo Express sert surtout à contrôler rapidement l'état de la base : collections, documents, compteurs et petites modifications ponctuelles.

Pour les imports du cours, on utilise `mongoimport` en ligne de commande. C'est plus fiable, répétable et documentable qu'une action faite dans une interface web.

docker compose exec -T mongodb mongoimport \
  --username root --password rootpass --authenticationDatabase admin \
  --db nyc_food --collection restaurants \
  --file /tmp/nyc-food/restaurants.json --jsonArray --drop

`mongoimport` est l'outil général à retenir pour importer du JSON, CSV ou TSV dans MongoDB.
