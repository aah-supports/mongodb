Analyser avec `aggregate`

`aggregate` sert à transformer des documents en résultats calculés : moyennes, classements, revenus, regroupements, jointures et collections de synthèse.

On l'utilise quand `find` ne suffit plus : calculs, regroupements, transformations de structure, jointures ou écriture d'un résultat matérialisé.

Comparer avec SQL

SQL

SELECT cuisine,
       AVG(overall_rating) AS average_rating,
       COUNT(*) AS restaurants
FROM restaurants
GROUP BY cuisine
ORDER BY average_rating DESC;

MongoDB

db.restaurants.aggregate([
  {
    $group: {
      _id: "$cuisine",
      average_rating: { $avg: "$ratings.overall" },
      restaurants: { $sum: 1 }
    }
  },
  { $sort: { average_rating: -1 } }
])

Le pipeline MongoDB exprime chaque étape explicitement : filtrer, transformer, grouper, trier.

Principe du pipeline

db.collection.aggregate([
  { stage1 },
  { stage2 },
  { stage3 }
])
ÉtapeQuestionStage typique
FiltrerQuels documents garder ?`$match`
FaçonnerQuels champs produire ?`$project`, `$set`
RegrouperQuels agrégats calculer ?`$group`
OrdonnerQuel classement afficher ?`$sort`, `$limit`
RelierQuelle collection joindre ?`$lookup`

L'ordre des étapes est important : MongoDB exécute le pipeline de haut en bas. Il n'existe pas un ordre universel obligatoire, mais l'ordre choisi peut changer le résultat.

Ce qui rend `aggregate` puissant, c'est le chaînage des étapes : on peut filtrer, transformer, regrouper, trier et même écrire le résultat final dans une collection, le tout dans une seule requête.

`$limit` puis `$sort`

db.restaurants.aggregate([
  { $limit: 10 },
  { $sort: { "ratings.overall": -1 } }
])

MongoDB prend 10 documents, puis trie seulement ces 10 documents.

`$sort` puis `$limit`

db.restaurants.aggregate([
  { $sort: { "ratings.overall": -1 } },
  { $limit: 10 }
])

MongoDB trie tous les restaurants, puis garde les 10 mieux notés.

En pratique, on place souvent `$match` tôt pour réduire le volume à traiter, puis on ajoute `$project`, `$set`, `$group`, `$sort`, `$limit` ou `$lookup` selon le résultat attendu.

Commencer par `$match`

`$match` filtre les documents. Il est souvent placé au début pour réduire le volume traité et permettre l'utilisation d'un index.

db.restaurants.aggregate([
  { $match: { "ratings.overall": { $gte: 22 } } }
])
// Équivalent proche en find
db.restaurants.find({ "ratings.overall": { $gte: 22 } })

Dans un pipeline analytique, `$match` joue le rôle du `WHERE` SQL.

Sélectionner et renommer avec `$project`

`$project` choisit les champs en sortie. Il peut aussi renommer ou calculer des valeurs simples.

db.restaurants.aggregate([
  {
    $project: {
      _id: 0,
      name: 1,
      cuisine: 1,
      price_for_two: 1,
      overall_rating: "$ratings.overall",
      average_detail_rating: {
        $round: [
          { $avg: ["$ratings.food", "$ratings.decor", "$ratings.service"] },
          1
        ]
      },
      value_score: {
        $round: [
          { $divide: ["$ratings.overall", "$price_for_two"] },
          3
        ]
      }
    }
  }
])
  • `overall_rating` renomme le champ imbriqué `ratings.overall`.
  • `average_detail_rating` calcule une moyenne à partir de `food`, `decor` et `service`.
  • `value_score` calcule un score note/prix en divisant la note globale moyenne par le prix estimé pour deux personnes.
  • `_id: 0` retire l'identifiant MongoDB de la sortie.

`$project` est utile quand on veut produire une sortie propre : quelques champs conservés, certains champs renommés, et des indicateurs calculés.

SQL comparable : `SELECT name, cuisine, price_for_two, ratings_overall AS overall_rating, ROUND((food + decor + service) / 3, 1) AS average_detail_rating, ROUND(ratings_overall / price_for_two, 3) AS value_score`.

Ajouter un champ avec `$set`

`$set` ajoute ou modifie un champ en conservant les autres. La différence avec `$project` est importante : `$project` redessine la sortie, alors que `$set` garde tout le document et ajoute seulement le champ calculé.

Dans un pipeline `aggregate`, ce champ est temporaire : il existe dans les documents produits par le pipeline, mais il n'est pas écrit dans la collection d'origine.

Pour modifier réellement les documents stockés, il faut utiliser une opération d'écriture comme `updateMany`, ou terminer le pipeline par une étape d'écriture comme `$merge` ou `$out`.

db.restaurants.aggregate([
  {
    $set: {
      value_score: {
        $round: [
          { $divide: ["$ratings.overall", "$price_for_two"] },
          3
        ]
      }
    }
  },
  { $sort: { value_score: -1 } },
  { $limit: 10 }
])

Ici, on calcule un score note/prix avant de trier les restaurants.

Regrouper avec `$group`

`$group` est l'équivalent de `GROUP BY`. Le champ `_id` définit la clé de regroupement.

db.restaurants.aggregate([
  {
    $group: {
      _id: "$cuisine",
      average_food: { $avg: "$ratings.food" },
      average_decor: { $avg: "$ratings.decor" },
      average_service: { $avg: "$ratings.service" },
      average_overall: { $avg: "$ratings.overall" },
      restaurants: { $sum: 1 }
    }
  },
  { $sort: { average_overall: -1 } }
])
AccumulateurUsage
`$sum`Compter ou additionner.
`$avg`Calculer une moyenne.
`$min` / `$max`Extraire une borne.
`$push`Construire un tableau.
`$addToSet`Construire un tableau sans doublons.

`$push` ou `$addToSet` ?

`$push` garde toutes les valeurs, y compris les doublons.

db.restaurants.aggregate([
  {
    $group: {
      _id: "$price_tier",
      cuisines: { $push: "$cuisine" }
    }
  }
])

`$addToSet` garde chaque valeur une seule fois.

db.restaurants.aggregate([
  {
    $group: {
      _id: "$price_tier",
      cuisines: { $addToSet: "$cuisine" }
    }
  }
])

Regrouper sur deux champs :

db.restaurants.aggregate([
  {
    $group: {
      _id: {
        cuisine: "$cuisine",
        price_tier: "$price_tier"
      },
      restaurant_nb: { $sum: 1 }
    }
  },
  { $sort: { restaurant_nb: -1 } }
])

Ici, MongoDB crée un groupe pour chaque combinaison `cuisine` + `price_tier`. Un groupe comme `{ cuisine: "Italian", price_tier: "$$" }` est distinct de `{ cuisine: "Italian", price_tier: "$$$" }`.

Le résultat montre donc, par exemple, combien de restaurants italiens existent dans chaque gamme de prix.

Éclater un tableau avec `$unwind`

On utilise `$unwind` quand on veut analyser les éléments d'un tableau individuellement.

Exemple : un restaurant peut avoir plusieurs tags. Si l'on veut compter les tags les plus fréquents, il faut d'abord transformer chaque tag en ligne séparée dans le pipeline.

`$unwind` transforme un document contenant un tableau en plusieurs documents, un par élément du tableau.

{
  name: "Demo Review Tags",
  tags: ["top_food", "great_service", "dinner"]
}

Après `{ $unwind: "$tags" }`, MongoDB produit temporairement trois documents dans le pipeline :

{ name: "Demo Review Tags", tags: "top_food" }
{ name: "Demo Review Tags", tags: "great_service" }
{ name: "Demo Review Tags", tags: "dinner" }

Le document d'origine n'est pas modifié. C'est seulement la forme des données qui circulent dans le pipeline qui change.

db.restaurants.aggregate([
  { $unwind: "$tags" },
  {
    $group: {
      _id: "$tags",
      occurrences: { $sum: 1 }
    }
  },
  { $sort: { occurrences: -1 } }
])
  1. `$unwind` crée une ligne temporaire par tag.
  2. `$group` regroupe ces lignes par valeur de tag.
  3. `$sum: 1` compte combien de fois chaque tag apparaît dans ces lignes temporaires.
  4. `$sort` classe les tags les plus fréquents en premier.

Le champ `occurrences` ne veut pas dire "nombre total de restaurants". Il signifie : nombre de documents produits par `$unwind` pour ce tag. Si un restaurant possède trois tags, il contribue à trois lignes temporaires, une par tag.

Sans `$unwind`, on compterait des tableaux entiers, pas les valeurs individuelles contenues dans ces tableaux.

Préparer un `$lookup`

`$lookup` relie deux collections dans un pipeline `aggregate`. Il ne s'utilise pas dans `find`.

Avant de l'écrire, identifier la collection de départ, la collection cible et les champs qui portent la relation.

ÉlémentQuestionExemple
DépartD'où part le pipeline ?`review_details`
CibleOù chercher l'information liée ?`restaurants`
Champ localQuel champ existe au départ ?`restaurant_id`
Champ cibleQuel champ correspond dans la cible ?`restaurant_id`
RésultatQuel champ ajouter ?`restaurant`

`as` ne désigne pas une collection existante. Il choisit le nom du nouveau champ ajouté dans chaque document produit par le pipeline.

as: "restaurant"

Avec ce nom, les données jointes seront accessibles ensuite avec des chemins comme `"$restaurant.name"` ou `"$restaurant.cuisine"`.

// Si on écrivait plutôt :
as: "restaurant_info"

// Il faudrait lire ensuite :
"$restaurant_info.name"
db.review_details.findOne(
  {},
  { _id: 0, restaurant_id: 1, rating: 1, sentiment: 1 }
)

db.restaurants.findOne(
  {},
  { _id: 0, restaurant_id: 1, name: 1, cuisine: 1 }
)

Faire un `$lookup` minimal

On commence petit : pas de `$unwind`, pas de projection complexe, pas de tri. L'objectif est d'observer la forme produite.

db.review_details.aggregate([
  { $limit: 3 },
  {
    $lookup: {
      from: "restaurants",
      localField: "restaurant_id",
      foreignField: "restaurant_id",
      as: "restaurant"
    }
  }
])

`$lookup` ajoute un champ `restaurant` parce que le pipeline contient `as: "restaurant"`. Ce champ est toujours un tableau, même quand une seule correspondance est attendue.

Filtrer puis utiliser `$unwind`

On place souvent `$match` avant `$lookup` pour ne joindre que les documents utiles.

db.review_details.aggregate([
  { $match: { verified_visit: true, rating: { $gte: 4.5 } } },
  {
    $lookup: {
      from: "restaurants",
      localField: "restaurant_id",
      foreignField: "restaurant_id",
      as: "restaurant"
    }
  },
  { $unwind: "$restaurant" },
  { $limit: 3 }
])

`$unwind: "$restaurant"` transforme le tableau `restaurant` en objet quand on attend un seul document relié.

// Avant
restaurant: [
  { restaurant_id: "NYC-ZAGAT-0001", name: "Daniella Ristorante" }
]

// Après $unwind
restaurant: {
  restaurant_id: "NYC-ZAGAT-0001",
  name: "Daniella Ristorante"
}

Projeter le résultat lié

On affiche la date de l'avis, sa note, son sentiment, puis le nom, la cuisine et le niveau de prix du restaurant lié.

db.review_details.aggregate([
  { $match: { verified_visit: true, rating: { $gte: 4.5 } } },
  {
    $lookup: {
      from: "restaurants",
      localField: "restaurant_id",
      foreignField: "restaurant_id",
      as: "restaurant"
    }
  },
  { $unwind: "$restaurant" },
  {
    $project: {
      _id: 0,
      reviewed_at: 1,
      rating: 1,
      sentiment: 1,
      name: "$restaurant.name",
      cuisine: "$restaurant.cuisine",
      price_tier: "$restaurant.price_tier"
    }
  },
  { $sort: { rating: -1, reviewed_at: -1 } },
  { $limit: 20 }
])

À retenir : `from` désigne la collection cible, `localField` le champ de départ, `foreignField` le champ correspondant, et `as` le champ ajouté.

Construire un indicateur métier complet

Question : quels restaurants combinent revenus importants et bonne satisfaction ?

db.orders.aggregate([
  { $match: { status: "paid" } },
  {
    $group: {
      _id: "$restaurant_id",
      revenue: { $sum: "$amount" },
      orders: { $sum: 1 },
      average_basket: { $avg: "$amount" }
    }
  },
  {
    $lookup: {
      from: "restaurants",
      localField: "_id",
      foreignField: "restaurant_id",
      as: "restaurant"
    }
  },
  { $unwind: "$restaurant" },
  { $match: { "restaurant.ratings.overall": { $gte: 22 } } },
  {
    $project: {
      _id: 0,
      name: "$restaurant.name",
      cuisine: "$restaurant.cuisine",
      overall_rating: "$restaurant.ratings.overall",
      revenue: { $round: ["$revenue", 2] },
      orders: 1,
      average_basket: { $round: ["$average_basket", 2] }
    }
  },
  { $sort: { revenue: -1 } },
  { $limit: 20 }
])

Matérialiser avec `$merge`

`$merge` écrit le résultat d'un pipeline dans une collection. C'est utile pour produire des KPI réutilisables.

KPI signifie Key Performance Indicator, en français indicateur clé de performance : chiffre d'affaires, nombre de commandes, panier moyen, note moyenne, etc.

db.orders.aggregate([
  { $match: { status: "paid" } },
  {
    $group: {
      _id: "$restaurant_id",
      revenue: { $sum: "$amount" },
      orders: { $sum: 1 },
      average_basket: { $avg: "$amount" }
    }
  },
  {
    $merge: {
      into: "restaurant_kpis",
      whenMatched: "replace",
      whenNotMatched: "insert"
    }
  }
])
ParamètreRôle
`into: "restaurant_kpis"`Écrit le résultat dans la collection `restaurant_kpis`.
`whenMatched: "replace"`Remplace le document existant si un document avec le même `_id` existe déjà.
`whenNotMatched: "insert"`Insère un nouveau document si aucun document correspondant n'existe.

Ici, le `_id` du résultat est le `restaurant_id` : relancer le pipeline met à jour les KPI existants et ajoute ceux des nouveaux restaurants.

Attention : `$merge` écrit réellement en base. On l'utilise quand le résultat doit être conservé.

Performance et ordre des stages

  • Mettre `$match` tôt pour réduire le volume.
  • Mettre `$project` tôt si certains gros champs sont inutiles.
  • Créer des index sur les champs filtrés au début du pipeline.
  • Éviter les `$lookup` trop larges sans filtre préalable.
  • Surveiller les tris et regroupements qui manipulent beaucoup de documents.
db.review_details.createIndex({ sentiment: 1, rating: -1 })
db.review_details.aggregate([
  { $match: { sentiment: "excellent", rating: { $gte: 4.5 } } },
  { $group: { _id: "$restaurant_id", reviews: { $sum: 1 } } },
  { $sort: { reviews: -1 } }
]).explain("executionStats")

Quand choisir `find` ou `aggregate` ?

QuestionOutilRaison
Afficher les restaurants français`find`Lecture directe de documents.
Afficher les restaurants triés par note`find`Filtre + tri simples.
Calculer une moyenne par cuisine`aggregate`Regroupement.
Compter les tags les plus fréquents`aggregate``$unwind` + `$group`.
Ajouter le nom du restaurant aux commandes`aggregate``$lookup`.
Créer une table de KPI`aggregate``$merge`.

Message clé : `aggregate` est le langage de transformation et d'analyse de MongoDB.
