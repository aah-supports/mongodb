Créer une base et comprendre le schéma

NiveauExemple
Base`nyc_food`
Collection`restaurants`
DocumentUn restaurant en BSON/JSON.

Création implicite

use demo_schema
db.restaurants.insertOne({
  name: "Demo Bistro",
  cuisine: "French",
  price_for_two: 48,
  ratings: { overall: 20.7 }
})

Une base et une collection existent physiquement après la première écriture.

Trois niveaux de schéma

Schéma implicite

Structure réellement présente dans les documents.

Schéma applicatif

Modèle attendu par l'application, la documentation et les requêtes.

Validation MongoDB

Règles JSON Schema appliquées par la base.

Imbriquer ou référencer ?

Imbriquer

Quand les données sont consultées ensemble et ne grossissent pas sans limite.

ratings: {
  food: 22,
  service: 20
}

Référencer

Quand les données sont nombreuses, évolutives ou consultées séparément.

{
  restaurant_id: "NYC-ZAGAT-0001",
  rating: 4.7
}
